## Implementation Summary

### Routes

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/workspaces` | Creates a workspace entity in Autumn under a parent customer |
| `POST` | `/workspaces/:workspaceId/use-credits` | Checks and atomically consumes workspace credits (check → track) |
| `GET` | `/workspaces/:workspaceId/balance` | Fetches the entity's full balance map from Autumn |
| `DELETE` | `/workspaces/:workspaceId` | Deletes a workspace entity (decrements seat/entity count) |

### Key Design Decisions

- **`POST /workspaces`** — Calls `autumn.entities.create()` with `featureId: 'workspaces'`, linking the workspace entity to the parent organization customer. Requires `workspaceId` and `customerId` in the request body.

- **`POST /workspaces/:workspaceId/use-credits`** — First calls `autumn.check()` with `entityId` to verify the workspace has sufficient balance. Returns `402` if insufficient. On success, calls `autumn.track()` to deduct credits. Accepts `featureId` (defaults to `'credits'`) and `value` (defaults to `1`).

- **`GET /workspaces/:workspaceId/balance`** — Calls `autumn.entities.get()` to retrieve the entity and returns its `balances` map. Accepts `customerId` and optional `featureId` query parameters.

- **`DELETE /workspaces/:workspaceId`** — Calls `autumn.entities.delete()` which also automatically decrements the entity usage counter. Requires `customerId` in the request body.

The `Autumn` client is initialized once at startup using `process.env.AUTUMN_SECRET_KEY`. The server listens on port 3000.
