### How it works

1. **Idempotency key** тАФ uses `process.env.IDEMPOTENCY_KEY` if set, otherwise generates a fresh `crypto.randomUUID()` on every run.

2. **Tracking** тАФ calls `autumn.track()` with:
   - `customerId: "user_test_001"`
   - `featureId: "api-calls"`
   - `value: 1`
   - `Idempotency-Key` HTTP header passed via `RequestOptions.headers`

3. **Duplicate protection** тАФ if Autumn returns HTTP 409 `duplicate_idempotency_key`, the error is caught gracefully. The script logs a clear message and exits cleanly without throwing, so callers know the event was already recorded and no double-billing occurred.

4. **Output** тАФ writes `{ idempotencyKey, result }` to `track_result.json` in the project directory.

### Usage

```bash
# Normal run тАФ generates a new UUID each time
node track_usage.js

# Idempotent retry тАФ same key is safe to repeat; second call is a no-op
IDEMPOTENCY_KEY=my-unique-request-id node track_usage.js
IDEMPOTENCY_KEY=my-unique-request-id node track_usage.js  # deduped, no double charge
```

### Key implementation detail

The `autumn-js` SDK's `TrackParams` type does not include an `idempotencyKey` field directly. Instead, the key is sent as the standard `Idempotency-Key` HTTP header via the second `RequestOptions` argument (which extends `RequestInit`, allowing custom headers). Autumn's server enforces deduplication by returning 409 for any repeated key within its deduplication window.
