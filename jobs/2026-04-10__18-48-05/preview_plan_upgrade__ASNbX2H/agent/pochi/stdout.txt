### `routes/billing.js`

An `Autumn` client is instantiated once at module load using `process.env.AUTUMN_SECRET_KEY`. Three routes are defined on an Express router:

| Route | Method | Behaviour |
|---|---|---|
| `/billing/preview-upgrade` | POST | Validates `customerId` + `targetPlanId`, calls `autumn.billing.previewAttach`, returns preview JSON (400 / 200 / 502) |
| `/billing/confirm-upgrade` | POST | Validates all three fields; short-circuits with `{ status: "cancelled" }` when `confirmed === false`; otherwise calls `autumn.attach` and returns `{ status: "upgraded", customerId, planId }` (400 / 200 / 502) |
| `/billing/batch-check` | POST | Fans out `autumn.customers.check` calls for every `customerId` in **parallel** via `Promise.all`, returns `{ results: [{ customerId, allowed }] }` |

**`confirmed` validation** uses `=== undefined` rather than a falsy check so that `false` is accepted as a valid value (and triggers the cancel branch) instead of being rejected as missing.
