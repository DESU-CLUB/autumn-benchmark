## Implementation Details

### 1. `middleware/auth.js`
- Exports `requirePremium` middleware
- Reads `x-customer-id` from request headers
- Returns HTTP 400 with `{ "error": "Missing customer ID" }` if absent
- Calls `autumn.customers.check({ customerId, featureId: 'premium-dashboard' })`
- Calls `next()` if `allowed` is `true`
- Returns HTTP 403 with `{ "error": "Premium plan required" }` if `allowed` is `false`

### 2. `routes/dashboard.js`
- Protected `GET /dashboard/premium` route
- Uses the `requirePremium` middleware
- Returns `{ "data": "Premium dashboard content", "customerId": "<customerId>" }` on success

### 3. `index.js`
- Mounts the dashboard router under `/dashboard`
- Starts the server on port 3000
- Includes JSON body parsing middleware
- Has a health check endpoint at `/`

### 4. Dependencies
- **express**: ^4.18.2
- **autumn-js**: ^1.0.0

## Usage

Start the server with:
```bash
cd /home/user/myproject
npm start
```

The server will run on port 3000 and use the `AUTUMN_SECRET_KEY` environment variable for Autumn SDK authentication.

## Example Requests

**Success (with premium access):**
```bash
curl -H "x-customer-id: customer123" http://localhost:3000/dashboard/premium
```

**Missing customer ID:**
```bash
curl http://localhost:3000/dashboard/premium
# Returns: { "error": "Missing customer ID" }
```

**No premium access:**
```bash
curl -H "x-customer-id: customer123" http://localhost:3000/dashboard/premium
# Returns: { "error": "Premium plan required" }
```
