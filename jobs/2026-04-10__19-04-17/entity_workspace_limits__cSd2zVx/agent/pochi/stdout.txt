I've implemented the multi-tenant workspace usage limits API using Express.js and the `autumn-js` SDK.

The Express server (`/home/user/myproject/index.js`) listens on port 3000 and exposes the following REST API endpoints:

- `POST /workspaces` — Creates a workspace entity in Autumn, passing `workspaceId`, `customerId`, `name`, and optionally `featureId` (defaults to 'credits').
- `POST /workspaces/:workspaceId/use-credits` — Uses `autumn.check` with `sendEvent: true` to atomically check and consume workspace credits. It first fetches the entity to identify the parent `customerId` before making the check.
- `GET /workspaces/:workspaceId/balance` — Fetches the entity details from Autumn and returns its current `balances` object.
- `DELETE /workspaces/:workspaceId` — Deletes the workspace entity from Autumn.

The app correctly passes the `AUTUMN_SECRET_KEY` environment variable to initialize the Autumn SDK client.
